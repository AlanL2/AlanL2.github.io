OOP

OOP stands for Object-Oriented Programming. This involves making classes in Java and using them as templates, so that multiple of the same thing can be created. Using a class as a template and creating something is called an Object .

So let's say if we were to make a Duck class. 

public class Duck{

}

All objects have what is known as Properties , and in Java, these are represented by the variables that are in the class, known as Instance Variables .
Here, we will have a variable representing if the duck is asleep or not, which can be represented by a boolean variable, and an age variable, representing the duck's age. 

public class Duck{
    private int age;
    private boolean asleep;
}

Creating a Constructor

To be able to create objects and differentiate objects from each other, we have to use a constructor to pass in values.


public class Duck{
    private int age;
    private boolean asleep;
    public Duck(int age, boolean asleep){
        this.age = age;
        this.asleep = asleep;
    }
}

Actions/Behaviours

An action or behaviour is something that your object is able to do. In our case, let's say that the duck will be able to quack and go to sleep, and vice versa. We can represent these by using void methods. The second and third method will change the boolean instance variable, so we call this a set method. 

public class Duck{
    private int age;
    private boolean asleep;
    public Duck(int age, boolean asleep){
        this.age = age;
        this.asleep = asleep;
    }
    public void quack(){
        System.out.println("Quack!");
    }
    public void goToSleep(){
        asleep = true;
    }
    public void wakeUp(){
        asleep = false;
    }
}

We can also have behaviours that check for something. These are called get methods. Thus, we will check if the duck is asleep or not, and this is relevant to the asleep variable.


public class Duck{
    private int age;
    private boolean asleep;
    public Duck(int age, boolean asleep){
        this.age = age;
        this.asleep = asleep;
    }
    public void quack(){
        System.out.println("Quack!");
    }
    public void goToSleep(){
        asleep = true;
    }
    public void wakeUp(){
        asleep = false;
    }
    public boolean getSleepStatus(){
        return asleep;
    }
}

Now, you are able to make the object do things. In our case, we can give the duck an age through the constructor, make it go to sleep or wake up, and get the sleep status of the duck to see if it is awake or asleep.

To create an object, we use the new keyword to create a new class, and use the same number of parameters as there are in the constructor. 

public static void main(String[] args) {
    Duck d = new Duck(10, false);
}

Encapsulation

Encapsulation is the hiding of information like the variables that we used in our Duck class. This prevents other classes from using the variables in our class. We have already done this, by using the private keyword with our variables. However, if we were to remove that keyword, then these variables would no longer be encapsulated.

Inheritance

This is when another class uses the similar template of a class. The original class is called the parent class, while the class using the template is called the child class. To do this, the child class must extend the parent class. This allows for the child class to by default take all of the methods of the parent class, and it is allowed to override those methods as well.
Let's say that there is a class called Animal that is the parent class, and it has its own encapsulated instance variables and its own methods. We can use our Duck class to extend that class, which takes its methods as default. 

public class Duck extends Animal{

}

Class Casting

Class casting involves the change of one class to another, either superclass to subclass or vice versa. Going from a subclass to a superclass is automatic, but if you are going the other way around, then you must manually do it. 

Duck d = (Duck)(new Animal());

Abstract Classes

These are classes that do not have any specific information in them. They only have methods, that can be overridden by subclasses. Thus, Abstract Classes are only used to be extended and overridden for the methods to do something specific.

public abstract class Animal{
    public abstract boolean getSleepStatus();
    public abstract boolean goToSleep();
}
private int age;
private boolean asleep;
public Duck(int age, boolean asleep){
    this.age = age;
    this.asleep = asleep;
}
public void quack(){
    System.out.println("Quack!");
}
public void goToSleep(){ //overriding the origial abstract method
    asleep = true;
}
public void wakeUp(){
    asleep = false;
}
public boolean getSleepStatus(){ //overriding the original abstract method
    return asleep;
}
